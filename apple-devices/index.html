Apple Devices

SMC & NVRAM, Wi-Fi dropping, 

Reset SMC or NVRAM
Check if MacBook / Air has an Intel processor or Apple Silicon (M1, M2, or M3 chips).at Apple Menu> About This Mac.

For Apple Silicon (M1, M2, M3) no traditional SMC or a manual NVRAM reset key combo. Hardware performs a self-check and refreshes these settings every time the computer is fully shut down.

Shut Down your Mac completely.
Close the lid for about 30 seconds.
Open the lid and turn it back on. This is the official equivalent of a reset for these models.

For Intel-based Macs with  Intel chip
1. NVRAM / PRAM Reset for settings like speaker volume, screen resolution, and startup-disk selection.

Shut down your Mac.
Turn it on and immediately press and hold these four keys together:
Option + Command + P + R for 20 seconds
Release after the second chime  OR
If it has the T2 Security Chip - Release after the Apple logo appears and disappears for the second time.

2. SMC Reset - for power management, fans, and Wi-Fi hardware communication  for models from 2018 to 2020 (with T2 Chip):

Shut down your Mac
On the built-in keyboard, press and hold:
Control (left side) + Option (left side) + Shift (right side)

Hold all three for 7 seconds, then also press and hold the Power button
Hold all four keys for another 7 seconds, then release
Wait a few seconds, then press the Power button to turn it back on

For older models (2017 and earlier): - Shut down Mac and ensure it's plugged into power.
On the left side of the keyboard, press Shift + Control + Option and the Power button at the same time.
Release all keys at once (sometimes the light on the charging cable will flicker
Press the Power button to restart

Common "Gotcha": If you use a firmware password, you must turn it off before these key combinations will work.

 

Wi-Fi dropping after connecting

Network configuration conflicts, DHCP lease expiration, or aggressive power-saving features

Renew the DHCP Lease
System Settings (or System Preferences) > Network
Select Wi-Fi & click Details (or Advanced) next to the Network
Click TCP/IP in the sidebar
Click Renew DHCP Lease, then click OK and Apply

Create a "New Network Location" rebuilding network preferences settings for Location
Network settings, look for the Locations dropdown (may need to click a "..." or "More" button
Choose Edit Locations... and click the [+] icon
Rename it Home,  City, etc > Done
Select this new location in the dropdown and click Apply

Clear System Configuration Files by deleting the configuration plists 
Turn Wi-Fi OFF.
Open Finder and press Cmd + Shift + G
Paste the path: /Library/Preferences/SystemConfiguration/
Find and delete (or move to a backup folder on your desktop) these files:

com.apple.airport.preferences.plist
com.apple.network.eapolclient.configuration.plist
com.apple.wifi.message-tracer.plist
NetworkInterfaces.plist
preferences.plist

Restart Mac and turn Wi-Fi back on

Adjust Router Band Settings - Mac might be trying to hop between 2.4 & 5 GHz  when the signal fluctuates, causing a drop.

The Fix: If the router uses the same name (SSID) for both, try giving them different names (e.g., "Home_WiFi" and "Home_WiFi_5G"). Connect the Mac specifically to the 5GHz band for better stability or the 2.4GHz if far from the router.

Check for Hardware Interference - USB 3.0 devices and poorly shielded hubs can actually emit radio frequency interference that kills Wi-Fi signals on MacBooks

If there are any dongles, external hard drives, or monitors plugged in, try unplugging them for 30 minutes to see if the Wi-Fi remains stable.

Use Wireless Diagnostics - open hidden tool to watch the drop happen in real-time
Hold the Option key and click the Wi-Fi icon in the menu bar
Select Open Wireless Diagnostics
Instead of just running the wizard, go to the top menu bar and select Window > Performance.

This will show a live graph of signal strength ($RSSI$) and noise. If you see a sudden spike in "Noise" right when it drops, there is external interference.

2. Create a "New Network Location"

Macs sometimes hold onto old, corrupted network preferences.3 Creating a fresh "Location" forces the OS to rebuild those settings from scratch.

3. Clear System Configuration Files

4. Adjust Router Band Settings

If you have a dual-band router (2.4GHz and 5GHz), your Mac might be trying to "hop" between them when the signal fluctuates, causing a drop.

The Fix: If your router uses the same name (SSID) for both, try giving them different names (e.g., "Home_WiFi" and "Home_WiFi_5G"). Connect your Mac specifically to the 5GHz band for better stability, or the 2.4GHz if you are far from the router.5







5. Check for Hardware Interference

USB 3.0 devices and poorly shielded hubs can actually emit radio frequency interference that kills Wi-Fi signals on MacBooks.6
